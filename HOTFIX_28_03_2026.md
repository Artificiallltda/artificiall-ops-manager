## 📋 Issues Corrigidos

### QA-C01: Funcionário registrado com telegram_id="pending"

**Problema:**  
O comando `/registrar` criava funcionários com `telegram_id="pending"`, que nunca era atualizado. Funcionários não conseguiam usar `/cheguei` porque o bot não os encontrava.

**Solução Implementada:**

1. **Novo comando `/register_me`:** Funcionários capturam seu próprio telegram_id
2. **Pending ID com username:** Registros usam `pending_{username}` como placeholder
3. **Métodos novos em `google_sheets.py`:**
   - `update_employee_telegram_id(pending_id, telegram_id)` - Atualiza ID pendente
   - `get_employee_by_pending_id(pending_id)` - Busca registro pendente

**Fluxo Atualizado:**
```
1. Admin: /registrar @joao João Silva +5511999998888 Desenvolvedor
   → Cria registro: telegram_id="pending_joao"

2. Funcionário: /register_me
   → Atualiza: "pending_joao" → "123456789" (telegram_id real)

3. Funcionário: /cheguei
   → Funciona! Bot encontra registro pelo telegram_id real
```

**Arquivos Modificados:**
- `handlers/register.py` - Adicionado `handle_register_me()`
- `integrations/google_sheets.py` - Adicionados métodos de atualização
- `handlers/__init__.py` - Exportado `handle_register_me`
- `bot.py` - Registrado handler `/register_me`

### QA-C02: Log de decisão expõe texto sensível

**Problema:**  
Decisões executivas eram logadas com texto completo no `operations.log`, expondo informações confidenciais (salários, demissões, estratégias).

**Solução Implementada:**

Removido `decision_text` do log. Mantido apenas metadados:

```python
# handlers/decision.py (atualizado)
op_logger.log_critical(
    command="decisao",
    telegram_id=telegram_id,
    user_name="Gean Santos",
    message="Executive decision registered",
    details={
        "decision_id": decision.id,
        "categoria": categoria or "Geral",
        "ceo_telegram_id": telegram_id,
        "text_length": len(decisao_texto),  # Apenas tamanho
        # decision_text NÃO é mais logado
    },
)
```

**Arquivos Modificados:**
- `handlers/decision.py` - Removido texto do log

### QA-C03: Timezone middleware pode falhar no Windows

**Problema:**  
`datetime.strptime("-03:00", "%z").tzinfo` pode falhar no Windows.

**Solução Implementada:**

Substituído por `timezone(timedelta(hours=-3))`:

```python
# middleware/logger.py (atualizado)
def _get_timestamp(self) -> str:
    """Get current timestamp in ISO 8601 format with timezone."""
    from datetime import timezone, timedelta

    # Fixed offset for America/Sao_Paulo (BRT = UTC-3)
    tz = timezone(timedelta(hours=-3))
    return datetime.now(tz).isoformat()
```

**Arquivos Modificados:**
- `middleware/logger.py` - Corrigido `_get_timestamp()`
